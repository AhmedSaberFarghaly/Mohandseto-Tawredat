# v1.0.0 release checklist

## Repository gate

- [ ] API Release build and full test suite pass.
- [ ] Empty-database migration integrity test passes.
- [ ] Admin lint, audit policy, and production build pass.
- [ ] Flutter analysis and tests pass.
- [ ] API/admin container builds and Compose validation pass in CI.
- [ ] Secret scan and dependency vulnerability scans pass or have approved, documented exceptions.
- [ ] Changelog, deployment notes, permissions matrix, threat model, OpenAPI, and rollback steps are current.

## Staging gate

- [ ] Exact release commit/image digest deployed with production-like TLS and persistent volumes.
- [ ] Automated authenticated release smoke passes.
- [ ] Critical customer/admin E2E scenarios pass.
- [ ] Backup and isolated restore rehearsal pass, including Data Protection keys.
- [ ] Performance, accessibility, RTL/responsive, device/browser, and provider sandbox checks are approved.
- [ ] Google and Microsoft OAuth pass physical Android/iOS sign-in, cancel, linking, repeat-login and 2FA scenarios.
- [ ] Monitoring and on-call alert delivery are verified.

## Production authorization

- [ ] Domain, hosting, secret store, provider live credentials, privacy/legal copy, and licensed commercial assets are supplied.
- [ ] Release, security, data, and business owners approve the change window and rollback point.
- [ ] Database and matching key backup is verified off host.
- [ ] Post-deploy smoke and observation window pass.
- [ ] Only then create the signed `v1.0.0` tag and release notes.

Unchecked external items are real launch blockers, not repository defects. Do not mark GA complete or create the tag before they are evidenced.
