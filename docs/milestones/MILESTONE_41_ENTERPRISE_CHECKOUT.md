# Milestone 41% — Enterprise Checkout

اكتملت رحلة Checkout المؤسسية للشاشات 123–151 مع الحفاظ على عزل الشركات وتثبيت بيانات الطلب لحظة الإنشاء.

## المنجز

- مراكز تكلفة بميزانية ومبلغ مستخدم ومحجوز وحد موافقة.
- ربط الطلب بمشروع وإدارة طالبة وملاحظات ومرجع داخلي.
- إضافة عنوان شركة جديد مع إحداثيات ومحدد موقع داخل التطبيق.
- رفع أمر شراء PDF/PNG/JPG بحد 10MB ومسار تنزيل معزول حسب Tenant.
- شحن قياسي وسريع واستلام، مع السماح بتقسيم التوصيل.
- دفع إلكتروني tokenized بمحاولات idempotent وحالات رفض وإعادة محاولة.
- دفع جزئي موزع بدقة بين الحد الائتماني والبطاقة.
- مراجعة ميزانية، حجز المبلغ للطلب المنتظر، وتثبيت كل بيانات السياق على Order.
- استمرار بوابة المنتجات المخصصة: لا يبدأ الإنتاج قبل Checkout والموافقة المطلوبة.

## التحقق

- 25 اختبار Backend ناجحًا، منها تجاوز الميزانية وحجزها، عزل مرفق PO، وفشل/إعادة محاولة/نجاح الدفع الجزئي.
- 7 اختبارات Flutter و`flutter analyze` بلا مشاكل.
- migration `EnterpriseCheckout` مطبقة بنجاح على قاعدة SQLite فارغة.
- لا توجد علاقة EF shadow لمحاولات الدفع، وتم التحقق من FK الفعلي إلى جلسة Checkout.
